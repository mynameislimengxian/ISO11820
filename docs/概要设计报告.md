## 1. 系统架构设计

### 1.1 分层架构

系统采用经典的四层架构，严格遵循"上层依赖下层，下层不感知上层"的原则：

```
┌──────────────────────────────────────────────────┐
│                   Forms 层（界面）                 │
│  LoginForm │ MainForm │ NewTestForm │ ...         │
│  负责：用户交互、数据展示、输入验证                  │
├──────────────────────────────────────────────────┤
│                  Core 层（业务核心）                │
│  AppGlobal │ TestController │ TestState           │
│  负责：全局状态管理、试验状态机、仿真引擎、事件广播   │
├──────────────────────────────────────────────────┤
│         Data 层              Services 层           │
│  DbHelper │ Models │    │ CSV/Excel/PDF 导出      │
│  负责：数据库操作、实体定义  │ 文件生成、报告输出     │
├──────────────────────────────────────────────────┤
│              外部依赖（NuGet 包）                   │
│  Sqlite │ OxyPlot │ EPPlus │ PDFsharp │ Serilog   │
└──────────────────────────────────────────────────┘
```

### 1.2 各层职责

| 层 | 命名空间 | 负责人 | 职责 |
|----|---------|:---:|------|
| Forms | `ISO11820.Forms` | C（廖紫彤）+ D（龚小倩） | 所有 UI 窗体、控件布局、用户交互响应、跨线程 UI 更新 |
| Core | `ISO11820.Core` | A（李孟鲜） | 全局单例配置读取、试验状态机流转、5 通道温度仿真、数据广播事件 |
| Data | `ISO11820.Data` | B（兰竣雯） | 数据库初始化建表、全部 CRUD 操作封装、实体类定义 |
| Services | `ISO11820.Services` | B（兰竣雯）+ D（龚小倩） | CSV 导出、Excel 导出（B）、PDF 导出、文件存储管理、测试框架（D） |

### 1.3 层间依赖关系

```
Forms ──依赖──→ Core（AppGlobal、TestController、DataBroadcast 事件）
Forms ──依赖──→ Data（DbHelper）
Forms ──依赖──→ Services（导出服务）

Core ──依赖──→ Data（DbHelper — 试验数据读写）

Data ──依赖──→ NuGet（Microsoft.Data.Sqlite、Microsoft.Extensions.Configuration）
Services ──依赖──→ NuGet（EPPlus、PDFsharp）
```

**关键规则**：
- 下层**绝不**引用上层命名空间
- 数据从下层传到上层通过**事件（event）**，不能直接调用 UI 方法
- 所有 UI 更新必须在 UI 线程执行（跨线程用 `Control.Invoke()`）

### 2.2 模块职责详述

**M1 全局上下文（AppGlobal）**
- 单例模式，程序启动时初始化
- 读取 `appsettings.json` 配置文件
- 提供全局配置属性：`DbPath`、`BaseDirectory`、`TargetFurnaceTemp`、`HeatingRatePerSecond` 等
- 其他模块通过 `AppGlobal.Instance` 访问

**M2 试验控制器（TestController）**
- 管理 5 状态试验状态机：Idle → Preparing → Ready → Recording → Complete
- 每 800ms 执行一次仿真 tick，更新 5 通道温度
- 分阶段温度仿真算法（升温、稳定、记录、降温）
- 自动判定温度稳定和状态转移
- 通过 DataBroadcast 事件向 UI 推送数据

**M3 数据广播（DataBroadcast）**
- `DataBroadcastEventArgs`：携带 5 通道温度、当前状态、计时秒数、系统消息
- `MasterMessage`：消息时间 + 内容 + 类型
- 事件在后台线程触发，UI 订阅者必须用 `Invoke` 切回 UI 线程

**M4 数据库初始化（DatabaseInitializer）**
- 程序首次运行时自动执行
- 创建 6 张表 + 5 条索引
- 插入初始数据：operators 2 条、apparatus 1 条、sensors 17 条
- 使用 `IF NOT EXISTS` 保证幂等性

**M5 数据库操作（DbHelper）**
- 封装所有 SQLite 操作，对外提供 16 个方法
- 全部使用参数化 SQL（`$param`），防止 SQL 注入
- 每个方法独立创建和释放连接，线程安全
- 方法包括：登录验证、设备读写、样品 CRUD、试验 CRUD、传感器读写、校准记录 CRUD

**M6 实体模型（Models）**
- 6 个 POCO 类，对应 6 张数据库表
- `Operator`、`Apparatus`、`ProductMaster`、`TestMaster`（43 个字段）、`Sensor`、`CalibrationRecord`
- `TestSummary`：查询结果摘要 DTO（用于列表展示）

**M11~M14 导出服务**
- CSV：试验完成自动生成，每秒一行 6 列温度数据
- Excel：三个 Sheet（试验信息 + 温度数据 + 嵌入图表），使用 EPPlus 7.4.1
- PDF：试验概要 + 曲线图 + 判定结论，使用 PDFsharp-MigraDoc 6.1.1
- 文件管理：自动创建目录、统一路径生成

## 4. 数据库设计

### 4.1 数据库选型

选用 **SQLite** 作为数据库引擎。SQLite 是嵌入式关系型数据库，数据库文件（`.db`）随程序一起存储，无需安装数据库服务。数据库文件路径由 `appsettings.json` 的 `Database.SqlitePath` 配置，默认 `Data\ISO11820.db`（相对程序根目录）。

### 4.2 表结构设计

系统共 6 张表：

| 表名 | 主键 | 记录数 | 说明 |
|------|------|:---:|------|
| `operators` | 无主键 | 2（初始） | 操作员账号，密码明文存储 |
| `apparatus` | `apparatusid` (INTEGER) | 1（初始） | 试验设备信息 |
| `productmaster` | `productid` (TEXT) | 动态增长 | 样品信息 |
| `testmaster` | `(productid, testid)` 联合主键 | 动态增长 | 试验记录（核心表） |
| `sensors` | `sensorid` (INTEGER) | 17（初始） | 传感器通道配置 |
| `CalibrationRecords` | `Id` (TEXT, GUID) | 动态增长 | 校准历史记录 |

**关键设计要点**：

- `operators` 表无主键约束，登录按 `username + pwd` 查询，不使用 `userid`
- `testmaster` 联合主键 `(productid, testid)`，查询必须同时提供两个值
- `testmaster.productid` 外键引用 `productmaster.productid`，保证数据完整性
- `testmaster` 新建时统计字段全部填 0，试验完成后 UPDATE 回填
- `testmaster.flag = "10000000"` 表示试验记录已保存
- `CalibrationRecords` 表名 PascalCase，与其他表的小写命名不同
- `CalibrationRecords.TemperatureData` 存储 JSON 字符串，需手动序列化/反序列化
- 温度时序数据不存入数据库，而是写入独立 CSV 文件

## 6. 接口设计

### 6.1 全局单例 AppGlobal

**提供方**：Core 层（A）

**访问方式**：`AppGlobal.Instance`

**公开属性**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `Configuration` | `IConfiguration` | 原始配置对象 |
| `DbPath` | `string` | 数据库文件路径，默认 `"Data\ISO11820.db"` |
| `BaseDirectory` | `string` | 文件存储根目录，默认 `"D:\ISO11820"` |
| `ReportsDirectory` | `string` | 报告输出目录，默认 `"D:\ISO11820\Reports"` |
| `EnableSimulation` | `bool` | 是否启用仿真模式 |
| `TargetFurnaceTemp` | `double` | 目标炉温（°C），默认 750.0 |
| `HeatingRatePerSecond` | `double` | 升温速率（°C/s），默认 5.0 |
| `TempFluctuation` | `double` | 温度波动幅度（°C），默认 0.5 |
| `StableThreshold` | `double` | 稳定判定阈值（°C），默认 3.0 |

**使用示例**：

```csharp
var db = new DbHelper(AppGlobal.Instance.DbPath);
```

### 6.2 TestController API

**提供方**：Core 层（A）

**公开方法**：

| 方法 | 返回值 | 说明 |
|------|--------|------|

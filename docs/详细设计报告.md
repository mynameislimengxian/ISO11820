## 1. 核心模块详细设计

### 1.1 试验状态机

#### 1.1.1 状态定义

系统定义 5 个试验状态，由 `TestState` 枚举管理：

```csharp
public enum TestState
{
    Idle,       // 空闲 — 初始状态
    Preparing,  // 升温中 — 炉温上升中
    Ready,      // 就绪 — 温度稳定，等待记录
    Recording,  // 记录中 — 正在记录数据
    Complete    // 完成 — 试验结束，等待保存
}
```

#### 1.1.2 状态转移图

```
                    ┌──────────┐
         点击升温    │   Idle   │
     ┌─────────────→│   空闲   │←─────────────┐
     │              └────┬─────┘              │
     │                   │                    │
     ▼                   │ 点击升温            │ 点击停止升温
 ┌──────────┐            │                    │ (Preparing/Ready
 │Preparing │            │                    │  /Complete)
 │ 升温中   │            │                    │
 └────┬─────┘            │                    │
      │                  │                    │
      │ 温度稳定         │                    │
      │ (745~755°C)      │                    │
      ▼                  │                    │
 ┌──────────┐            │                    │
 │  Ready   │            │                    │
 │  就绪    │────────────┘                    │
 └────┬─────┘  温度跌出范围                   │
      │      自动回退 Preparing               │
      │                                       │
      │ 点击开始记录                           │
      ▼                                       │
 ┌──────────┐                                │
 │Recording │                                │
 │ 记录中   │                                │
 └────┬─────┘                                │
      │                                       │
      │ 到达终止条件 / 手动停止                 │
      ▼                                       │
 ┌──────────┐    保存试验记录                  │
 │ Complete │─────────────────→ Preparing      │
 │  完成    │    (保持炉温)                     │
 └──────────┘                                 │
      │                                       │
      │ 点击停止升温                            │
      └───────────────────────────────────────┘
```

#### 1.1.3 状态转移规则

| 转移 | 触发条件 | 动作 |
|------|---------|------|
| Idle → Preparing | 用户点击"开始升温" | 启动仿真 Timer（800ms），初始化温度 |
| Preparing → Ready | TF1 进入 745~755°C 且稳定计数器 > 3 | 系统提示"温度已稳定，可以开始记录" |
| Ready → Recording | 用户点击"开始记录" | 计时器清零并启动 |
| Recording → Complete | 到达终止条件或用户点击"停止记录" | 停止 Timer，记录试验结束 |
| Complete → Preparing | 用户保存试验记录 | 保持炉温，清空未保存标记 |
| Preparing/Ready/Complete → Idle | 用户点击"停止升温" | 停止 Timer，温度开始下降 |
| Ready → Preparing | 温度跌出 745~755°C | 自动回退，稳定计数器清零 |

#### 1.1.4 线程安全设计

所有状态转移操作由 `lock (_lock)` 保护，确保多线程并发访问安全：

```csharp
private readonly object _lock = new();

public bool StartHeating()
{
    lock (_lock)
    {
        if (State != TestState.Idle) return false;
        State = TestState.Preparing;
        // ...
        return true;
    }
}
```

### 1.2 温度仿真算法

#### 1.2.1 仿真周期

`System.Timers.Timer` 每 800ms 触发一次 tick，在 `OnTimerTick` 回调中执行：

1. 更新 5 通道温度
2. 检查状态转移
3. 计时更新
4. 广播数据到 UI

#### 1.2.2 升温阶段算法（Preparing）

```
TF1 += HeatingRatePerSecond × 0.8 + Random(-1, 1) × TempFluctuation
TF2 += HeatingRatePerSecond × 0.8 + Random(-1, 1) × TempFluctuation
TS   = TF1 × 0.3 + Random(-1, 1) × TempFluctuation
TC   = TF1 × 0.25 + Random(-1, 1) × TempFluctuation
TCal = TF1 + Random(-1, 1) × TempFluctuation × 2
```

**说明**：TF1/TF2 以每秒升温速率的 80% 逐 tick 上升，叠加 ±0.5°C 随机噪声。TS 和 TC 在非记录阶段保持低值跟随。

#### 1.2.3 稳定阶段算法（Ready）

```
TF1 = 750.0 + Random(-1, 1) × TempFluctuation
TF2 = 750.0 + Random(-1, 1) × TempFluctuation
TS   = TF1 × 0.3 + Random(-1, 1) × TempFluctuation
TC   = TF1 × 0.25 + Random(-1, 1) × TempFluctuation
TCal = TF1 + Random(-1, 1) × TempFluctuation × 2
```

**说明**：炉温直接钳位到 750°C，仅在 ±0.5°C 范围内随机波动。

#### 1.2.4 记录阶段算法（Recording）

```
TF1 = 750.0 + Random(-1, 1) × TempFluctuation
TF2 = 750.0 + Random(-1, 1) × TempFluctuation
TS  += [min(TF1 × 0.95, 800) - TS] × 0.02 + Random(-1, 1) × TempFluctuation
TC  += [min(TF1 × 0.85, 750) - TC] × 0.01 + Random(-1, 1) × TempFluctuation
TCal = TF1 + Random(-1, 1) × TempFluctuation × 2
```

**说明**：炉温保持稳定，表面温 TS 向 炉温×0.95 指数接近（系数 0.02），中心温 TC 向 炉温×0.85 指数接近（系数 0.01，更慢）。指数逼近公式模拟物理热传导过程。

#### 1.2.5 降温阶段算法（Complete）

```
TF1 -= 0.5 + |Random(-1, 1) × TempFluctuation| × 0.1
TF2 -= 0.5 + |Random(-1, 1) × TempFluctuation| × 0.1
TS   = TF1 × 0.5
TC   = TF1 × 0.4
TCal = TF1 + Random(-1, 1) × TempFluctuation × 2
```

**说明**：每 tick 下降约 0.5°C，表面和中心温度随炉温下降。

#### 1.2.6 随机噪声生成

```csharp
private double RandNoise()
{
    return (_rng.NextDouble() * 2 - 1) * _fluctuation;
}
```

`_fluctuation` 默认值 0.5，噪声范围 [-0.5, +0.5]°C。

### 1.3 温漂计算

#### 1.3.1 算法原理

温漂（Temperature Drift）指单位时间内温度变化的趋势量，使用 MathNet.Numerics 的线性回归计算。

**输入**：最近 600 个数据点（约 10 分钟的采样数据，按 800ms/tick 计算）

**算法**：

```csharp
using MathNet.Numerics;

// 取最近 600 个 TF1 温度值
double[] x = Enumerable.Range(0, count).Select(i => (double)i).ToArray();
double[] y = temperatureHistory.ToArray();

// 线性回归拟合 y = a + b*x
var (a, b) = SimpleRegression.Fit(x, y);

// 斜率 b 的单位是 °C/tick
// 转换为 °C/10min: b × (600 / ticksPer10Min)
double drift = b * (600.0 / count * 10 * 60 / 0.8);
```

**判定标准**：温漂绝对值 < 2°C/10min → 炉温稳定。

#### 1.3.2 边界处理

- 刚启动时数据点不足 600 个，使用已有数据点计算
- 数据点少于 10 个时不计算温漂，显示 "N/A"

### 1.4 终止条件判定

#### 1.4.1 标准 60 分钟模式

```
检查点：每 5 分钟检查一次（t = 1800, 2100, 2400, 2700, 3000, 3300 秒）
终止条件：10 分钟温漂有效，且 TF1/TF2 温漂均不超过 MaxTemperatureDriftPerTenMinutes
无条件终止：t = 3600 秒
```

#### 1.4.2 固定时长模式

```
终止条件：ElapsedSeconds >= TargetDurationSeconds
忽略提前终止检查
```

#### 1.4.3 手动终止

```
用户点击"停止记录"按钮：
  - 若已有有效记录样本 → 进入 Complete
  - 否则 → 回到 Preparing（保持恒温，等待重新记录）
```

#### 1.4.4 判定结论

试验完成后，根据以下规则判定"通过"或"不通过"：

```csharp
bool passed = deltatf <= 50
           && lostweight_per <= 50
           && flameduration < 5;
```

**保存流程**：

```
btnCreate_Click
  → 验证必填字段（样品编号、样品名称、质量等）
  → 构造 TestMaster 对象（统计字段全部填 0）
  → 调用 DbHelper.InsertTest(test)
  → 调用 DbHelper.InsertProduct(product)（如样品不存在）
  → 设置 TestController.CurrentProductId、CurrentOperator
  → 关闭对话框，返回主界面
```

### 3.4 试验现象记录窗体（RecordForm）

**控件清单**：

| 控件 | 属性 | 说明 |
|------|------|------|
| CheckBox | Text="是否出现持续火焰" | 勾选后启用火焰输入 |
| NumericUpDown × 2 | Enabled 绑定火焰 CheckBox | 火焰发生时刻（秒）、持续时间（秒） |
| NumericUpDown | 必填 | 试验后质量（g） |
| TextBox | Multiline | 备注（可选） |
| Label × 3 | 只读，自动计算 | 失重量、失重率、各通道温升 |
| Button × 2 | | 保存、取消 |

**保存流程**：

```
btnSave_Click
  → 验证：试验后质量必填
  → 自动计算：
      失重量 = preweight - postweight
      失重率 = 失重量 / preweight × 100%
      各通道温升 = 最终值 - 初始值
      判定项 deltatf = 表面温升（当前代码取 deltats）
  → 调用 DbHelper.UpdateTestResult(test)
     （写入所有统计字段 + flag = "10000000"）
  → 调用 TestController.MarkSaved()
  → 关闭对话框
```

### 3.5 数据流：按钮点击 → 状态机 → 数据广播 → UI 更新

```
用户点击"开始升温"
  → MainForm.btnStartHeating_Click()
  → TestController.StartHeating()
  → 状态: Idle → Preparing
  → Timer 启动（800ms）
  → 每 800ms 执行 OnTimerTick:
      1. UpdateTemperatures() — 更新 5 通道温度
      2. CheckStateTransition() — 检查状态转移
      3. 构造 DataBroadcastEventArgs
      4. 触发 DataBroadcast 事件（后台线程）
  → MainForm.OnDataBroadcast()（后台线程）
  → this.Invoke() 切回 UI 线程
  → 更新温度 Label、曲线图、计时器、状态标签、消息日志
```

| **停止记录** | ❌ | ❌ | ❌ | ✅ | ❌ |
| **参数设置** | ✅ | ✅ | ✅ | ❌ | ✅ |

// 判断是否存在已完成但未保存的试验
bool isBlocked = test != null

## 5. 导出格式规范

### 5.1 CSV 格式

**文件路径**：`{BaseDirectory}\TestData\{productid}\{testid}\sensor_data.csv`

**格式**：无标题行，逗号分隔，每秒一行

```csv
0,25.0,24.9,24.5,24.3,25.1
1,30.1,30.0,24.6,24.4,25.0
2,70.2,69.8,24.8,24.5,25.2
...
3600,750.2,749.8,620.5,480.1,751.0
```

**列定义**：

| 列序号 | 列名 | 对应通道 | 数据类型 | 精度 |
|:---:|------|---------|---------|------|
| 1 | Time | — | 整数 | 秒 |
| 2 | Temp1 | 炉温1（TF1） | 浮点数 | 1 位小数 |
| 3 | Temp2 | 炉温2（TF2） | 浮点数 | 1 位小数 |
| 4 | TempSurface | 表面温（TS） | 浮点数 | 1 位小数 |
| 5 | TempCenter | 中心温（TC） | 浮点数 | 1 位小数 |
| 6 | TempCalibration | 校准温（TCal） | 浮点数 | 1 位小数 |

**生成时机**：试验完成（Recording → Complete）时自动生成

### 5.2 Excel 格式

**文件路径**：`{ReportsDirectory}\{testid}_报告.xlsx`

**Sheet 1：试验信息表**

| 列 | 内容 |
|----|------|
| 试验编号 | {productid} / {testid} |
| 试验日期 | {testdate} |
| 样品名称 | {productname} |
| 操作员 | {operator} |
| 试验前质量 | {preweight} g |
| 试验后质量 | {postweight} g |
| 失重率 | {lostweight_per}% |
| 样品温升 | {deltatf}°C |
| 火焰持续时间 | {flameduration}s |
| 判定结论 | 通过 / 不通过 |

**Sheet 2：温度数据表**

将 CSV 内容完整导入，带标题行，数字格式化。

**Sheet 3：温度曲线图**

使用 EPPlus 图表功能，嵌入 4 条折线（TF1/TF2/TS/TC），X 轴时间，Y 轴温度。

**生成时机**：用户手动点击"导出 Excel"按钮

### 5.3 PDF 格式

**文件路径**：`{ReportsDirectory}\{testid}_报告.pdf`

# 数据需求文档 — ISO 11820 建筑材料不燃性试验仿真系统

> **版本**：v1.0  
> **目标读者**：开发人员（负责 DbHelper + SQL 建表 + 配置管理）  
> **数据引擎**：SQLite（本地文件，无需安装服务）  
> **文档目的**：明确系统中所有数据的结构、约束、关系及流向，作为数据库设计与编码的基准。

---

## 一、数据实体总览

系统共涉及 **6 张数据库表 + 1 类 CSV 文件 + 1 份配置文件**：

| 序号 | 实体 | 存储方式 | 说明 |
|------|------|---------|------|
| 1 | `operators` | SQLite 表 | 操作员/用户账号 |
| 2 | `apparatus` | SQLite 表 | 试验设备信息 |
| 3 | `productmaster` | SQLite 表 | 样品信息 |
| 4 | `testmaster` | SQLite 表 | 试验记录（**核心表**） |
| 5 | `sensors` | SQLite 表 | 传感器通道配置 |
| 6 | `CalibrationRecords` | SQLite 表 | 设备校准历史 |
| 7 | 温度时序数据 | CSV 文件 | 每次试验的逐秒温度数据 |
| 8 | `appsettings.json` | JSON 文件 | 全局配置 |

## 二、各实体详细定义

### 2.1 operators（操作员表）

**用途**：存储系统登录账号。登录验证按 `username + pwd` 匹配，不使用 `userid`。

| 序号 | 字段名 | 类型 | 约束 | 说明 |
|------|--------|------|------|------|
| 1 | `userid` | TEXT | NOT NULL | 用户ID（当前初始化为 "1"、"2"），**不用于登录校验** |
| 2 | `username` | TEXT | NOT NULL | 登录用户名。管理员="admin"，试验员="experimenter" |
| 3 | `pwd` | TEXT | NOT NULL | 明文密码，默认 "123456" |
| 4 | `usertype` | TEXT | NOT NULL | 角色：`admin`（管理员）或 `operator`（试验员） |

**⚠️ 特殊说明**：
- 此表 **无主键约束**（当前代码未设 PRIMARY KEY）。
- 密码 **明文存储**。
- 登录查询条件：`WHERE username = ? AND pwd = ?`。

**初始数据**：

| userid | username | pwd | usertype |
|--------|----------|-----|----------|
| 1 | admin | 123456 | admin |
| 2 | experimenter | 123456 | operator |

### 2.2 apparatus（设备表）

**用途**：存储试验炉设备信息。新建试验时自动带入设备编号、名称、检定日期、恒功率值。

| 序号 | 字段名 | 类型 | 约束 | 说明 |
|------|--------|------|------|------|
| 1 | `apparatusid` | INTEGER | **PK** | 设备自增ID |
| 2 | `innernumber` | TEXT | NOT NULL | 设备内部编号，如 "FURNACE-01" |
| 3 | `apparatusname` | TEXT | NOT NULL | 设备名称，如 "一号试验炉" |
| 4 | `checkdatef` | date | NOT NULL | 检定有效期开始日期 |
| 5 | `checkdatet` | date | NOT NULL | 检定有效期结束日期 |
| 6 | `pidport` | TEXT | NOT NULL | PID 控制器串口，如 "COM9" |
| 7 | `powerport` | TEXT | NOT NULL | 功率控制器串口，如 "COM9" |
| 8 | `constpower` | INTEGER | NULL | 上次记录的恒功率值（0~25600），可空 |

**初始数据**：

| apparatusid | innernumber | apparatusname | checkdatef | checkdatet | pidport | powerport | constpower |
|-------------|-------------|---------------|------------|------------|---------|-----------|------------|
| 0 | FURNACE-01 | 一号试验炉 | date('now') | date('now', '+1 year') | COM9 | COM9 | 2048 |

| 序号 | 字段名 | 类型 | 约束 | 说明 |

| 6 | `flag` | TEXT | NULL | 备用标记字段 |

### 2.4 testmaster（试验记录表）⭐ 核心表

**用途**：存储每次试验的完整信息。**主键为 `(productid, testid)` 联合主键**。  
**外键**：`productid` → `productmaster.productid`。

**试验生命周期与字段填充时机**：

```
新建试验（INSERT）
  → 基本信息 + 质量 + 设备信息 已填充
  → 统计字段（温度、温升、时长等）全部填 0

试验完成（UPDATE）
  → 填入：postweight, lostweight, lostweight_per, totaltesttime
  → 填入：各通道 max/final/delta, constpower, phenocode
  → flag = "10000000"
```

#### 基本信息（试验开始即写入）

| 序号 | 字段名 | 类型 | 约束 | 说明 |
|------|--------|------|------|------|
| 1 | `productid` | TEXT | **PK, FK** | 样品编号，关联 productmaster |
| 2 | `testid` | TEXT | **PK** | 试验标识，格式 `yyyyMMdd-HHmmss` |
| 3 | `testdate` | date | NOT NULL | 试验日期 |
| 4 | `ambtemp` | REAL | NOT NULL | 环境温度（°C） |
| 5 | `ambhumi` | REAL | NOT NULL | 环境湿度（%） |
| 6 | `according` | TEXT | NOT NULL | 试验依据标准，如 "ISO 11820:2022" |
| 7 | `operator` | TEXT | NOT NULL | 操作员用户名 |
| 8 | `apparatusid` | TEXT | NOT NULL | 设备编号 |
| 9 | `apparatusname` | TEXT | NOT NULL | 设备名称（冗余，避免关联查询） |
| 10 | `apparatuschkdate` | date | NOT NULL | 设备检定日期 |
| 11 | `rptno` | TEXT | NOT NULL | 报告编号 |

#### 质量数据（新建时 preweight 写入，其余完成时 UPDATE）

| 序号 | 字段名 | 类型 | 约束 | 说明 |
|------|--------|------|------|------|
| 12 | `preweight` | REAL | NOT NULL | 试验前样品质量（g） |
| 13 | `postweight` | REAL | NOT NULL | 试验后样品质量（g），初始填 0 |
| 14 | `lostweight` | REAL | NOT NULL | 失重量 = preweight - postweight，初始填 0 |
| 15 | `lostweight_per` | REAL | NOT NULL | **【判定项】** 失重率（%）= lostweight / preweight × 100，初始填 0 |

#### 试验过程数据（完成时 UPDATE）

| 序号 | 字段名 | 类型 | 约束 | 说明 |
|------|--------|------|------|------|
| 16 | `totaltesttime` | INTEGER | NOT NULL | 总试验时长（秒），初始填 0 |
| 17 | `constpower` | INTEGER | NOT NULL | 恒功率值（0~25600），初始填 0 |
| 18 | `phenocode` | TEXT | NOT NULL | 现象编码（勾选项序列化字符串），初始填 "" |
| 19 | `flametime` | INTEGER | NOT NULL | 火焰开始时刻（秒），无火焰填 0 |
| 20 | `flameduration` | INTEGER | NOT NULL | 火焰持续时间（秒），无火焰填 0 |

#### 各通道温度最大值（完成时 UPDATE）

| 序号 | 字段名 | 类型 | 约束 | 说明 |
|------|--------|------|------|------|
| 21 | `maxtf1` | REAL | NOT NULL | 炉温1 最大值（°C） |
| 22 | `maxtf2` | REAL | NOT NULL | 炉温2 最大值（°C） |
| 23 | `maxts` | REAL | NOT NULL | 表面温 最大值（°C） |
| 24 | `maxtc` | REAL | NOT NULL | 中心温 最大值（°C） |
| 25 | `maxtf1_time` | INTEGER | NOT NULL | 炉温1 最大值时刻（秒） |
| 26 | `maxtf2_time` | INTEGER | NOT NULL | 炉温2 最大值时刻（秒） |
| 27 | `maxts_time` | INTEGER | NOT NULL | 表面温 最大值时刻（秒） |
| 28 | `maxtc_time` | INTEGER | NOT NULL | 中心温 最大值时刻（秒） |

#### 各通道温度最终值（试验结束时刻，完成时 UPDATE）

| 序号 | 字段名 | 类型 | 约束 | 说明 |
|------|--------|------|------|------|
| 29 | `finaltf1` | REAL | NOT NULL | 最终炉温1（°C） |
| 30 | `finaltf2` | REAL | NOT NULL | 最终炉温2（°C） |
| 31 | `finalts` | REAL | NOT NULL | 最终表面温（°C） |
| 32 | `finaltc` | REAL | NOT NULL | 最终中心温（°C） |
| 33 | `finaltf1_time` | INTEGER | NOT NULL | 最终炉温1 对应时刻（秒） |
| 34 | `finaltf2_time` | INTEGER | NOT NULL | 最终炉温2 对应时刻（秒） |
| 35 | `finalts_time` | INTEGER | NOT NULL | 最终表面温 对应时刻（秒） |
| 36 | `finaltc_time` | INTEGER | NOT NULL | 最终中心温 对应时刻（秒） |

#### 温升数据（完成时 UPDATE）

| 序号 | 字段名 | 类型 | 约束 | 说明 |
|------|--------|------|------|------|
| 37 | `deltatf1` | REAL | NOT NULL | 炉温1 温升 = finaltf1 - 环境温度 |
| 38 | `deltatf2` | REAL | NOT NULL | 炉温2 温升 = finaltf2 - 环境温度 |
| 39 | `deltatf` | REAL | NOT NULL | **【判定项】** 样品温升（°C），当前代码取表面温升 deltats |
| 40 | `deltats` | REAL | NOT NULL | 表面温 温升 = finalts - 环境温度 |
| 41 | `deltatc` | REAL | NOT NULL | 中心温 温升 = finaltc - 环境温度 |

#### 备注与标记

| 序号 | 字段名 | 类型 | 约束 | 说明 |
|------|--------|------|------|------|
| 42 | `memo` | TEXT | NULL | 备注（可选） |
| 43 | `flag` | TEXT | NULL | 完成标记。`"10000000"` = 试验记录已保存。**此标记用于控制 UI 按钮状态** |

#### 索引

| 索引名 | 字段 | 用途 |
|--------|------|------|
| `IX_Testmaster_Testdate` | `testdate` | 按日期查询历史记录 |
| `IX_Testmaster_Operator` | `operator` | 按操作员筛选 |
| `IX_Testmaster_Testdate_Productid` | `testdate, productid` | 组合查询 |

#### 关键业务规则

| 规则 | 说明 |
|------|------|
| **联合主键** | 查询一条试验记录必须同时提供 `productid` + `testid` |
| **flag = "10000000"** | 表示试验记录已保存。未保存时用户不能新建试验或重新开始记录 |
| **初始 INSERT** | 新建时除基本信息外，所有统计字段填 0 或 "" |
| **完成 UPDATE** | 试验结束后一次性更新所有统计字段 |
| **totaltesttime > 0 且 flag ≠ "10000000"** | 存在"已完成但未保存"的试验。UI 应阻止新建试验 |

### 2.5 sensors（传感器配置表）

**用途**：存储传感器通道的元数据配置。首次运行时写入，运行时动态更新 `outputvalue` 和 `inputvalue`。

| 序号 | 字段名 | 类型 | 约束 | 说明 |
|------|--------|------|------|------|
| 1 | `sensorid` | INTEGER | **PK** | 传感器通道编号 |
| 2 | `sensorname` | TEXT | NOT NULL | 传感器代号，如 "Sensor0", "Sensor1" |
| 3 | `dispname` | TEXT | NOT NULL | 显示名称，如 "炉温1" |
| 4 | `sensorgroup` | TEXT | NOT NULL | 分组标识："采集" 或 "校准" |
| 5 | `unit` | TEXT | NOT NULL | 单位，如 "℃" |
| 6 | `discription` | TEXT | NOT NULL | 描述文字 |
| 7 | `flag` | TEXT | NOT NULL | 标记，初始化为 "启用" |
| 8 | `signalzero` | REAL | NOT NULL | 信号零点 |
| 9 | `signalspan` | REAL | NOT NULL | 信号量程 |
| 10 | `outputzero` | REAL | NOT NULL | 输出温度下限（如 0） |
| 11 | `outputspan` | REAL | NOT NULL | 输出温度上限（如 1000） |
| 12 | `outputvalue` | REAL | NOT NULL | 当前温度值（**运行时每秒更新**） |
| 13 | `inputvalue` | REAL | NOT NULL | 当前输入值（**运行时每秒更新**） |
| 14 | `signaltype` | INTEGER | NOT NULL | 信号类型：`4` = 数字量（仿真模式用） |

**初始数据（5 个业务通道 + 12 个备用通道，共 17 条）**：

| 4~15 | Sensor4~15 | 备用通道5~16 | 采集 | 预留扩展 |

### 2.7 温度时序数据（CSV 文件）

**用途**：存储每次试验的逐秒温度原始数据，**不入库**，存为独立 CSV 文件。

**存储路径规则**：
```
{FileStorage.BaseDirectory}\TestData\{productid}\{testid}\sensor_data.csv
```
默认值：`D:\ISO11820\TestData\{productid}\{testid}\sensor_data.csv`

**文件结构**（每秒一行，无标题行，共 6 列）：
```csv
Time,Temp1,Temp2,TempSurface,TempCenter,TempCalibration
0,25.0,24.9,24.5,24.3,25.1
1,30.1,30.0,24.6,24.4,25.0
...
3600,750.2,749.8,620.5,480.1,751.0
```

| 列名 | 对应通道 | 单位 | 说明 |
|------|---------|------|------|
| Time | — | 秒 | 记录序号，从 0 开始 |
| Temp1 | 炉温1（TF1） | °C | 保留 1 位小数 |
| Temp2 | 炉温2（TF2） | °C | 保留 1 位小数 |
| TempSurface | 表面温（TS） | °C | 保留 1 位小数 |
| TempCenter | 中心温（TC） | °C | 保留 1 位小数 |
| TempCalibration | 校准温（TCal） | °C | 保留 1 位小数 |

**数据产生频率**：每 800ms 一个采样周期，但 CSV 按 **每秒一行** 写入。

**读取方**：报告生成模块（Excel/PDF 导出时读取 CSV 绘制曲线图）。

## 三、配置文件结构（appsettings.json）

**存储路径**：应用程序根目录 `appsettings.json`。

**读取方式**：通过 `Microsoft.Extensions.Configuration` 读取。

### 3.1 配置节一览

| 配置节 | 用途 | 使用方 |
|--------|------|--------|
| `Database` | 数据库路径和提供程序 | DbHelper 初始化 |
| `Hardware` | 硬件默认参数 | 仿真引擎、设备初始化 |
| `Simulation` | 仿真行为参数 | 传感器仿真器、试验控制器 |
| `FileStorage` | 文件存储路径 | CSV 写入、报告导出 |
| `Report` | 报告输出配置 | PDF/Excel 导出服务 |

### 3.2 各配置项详细说明

#### Database

| 键 | 类型 | 默认值 | 说明 |
|----|------|--------|------|
| `Provider` | string | `"Sqlite"` | 数据库提供程序类型 |
| `SqlitePath` | string | `"Data\\ISO11820.db"` | SQLite 文件相对路径（相对 exe 所在目录） |

#### Hardware

| 键 | 类型 | 默认值 | 说明 |
|----|------|--------|------|
| `ConstPower` | int | `2048` | 默认恒功率值（0~25600） |
| `PidTemperature` | int | `750` | PID 目标温度（°C） |
| `SensorProtocol` | string | `"ModbusRtu"` | 传感器通信协议（仿真模式不使用） |

#### Simulation

| 键 | 类型 | 默认值 | 说明 |
|----|------|--------|------|
| `EnableSimulation` | bool | `true` | **总开关**：是否启用仿真模式 |
| `SimulateSensors` | bool | `true` | 是否仿真传感器数据 |
| `SimulatePidController` | bool | `true` | 是否仿真 PID 控制器 |
| `InitialFurnaceTemp` | double | `720.0` | 炉温初始值（°C），演示用从 720°C 起步 |
| `TargetFurnaceTemp` | double | `750.0` | 目标炉温（°C），ISO 标准要求 |
| `HeatingRatePerSecond` | double | `40.0` | 仿真升温速率（°C/s） |
| `TempFluctuation` | double | `0.5` | 温度随机噪声幅度（°C） |
| `StableThreshold` | double | `3.0` | 稳定判定阈值：目标温度 ±3°C 视为稳定 |
| `SimulateFlame` | bool | `false` | 是否仿真火焰检测 |

#### FileStorage

| 键 | 类型 | 默认值 | 说明 |
|----|------|--------|------|
| `BaseDirectory` | string | `"D:\\ISO11820"` | 数据存储根目录 |
| `TestDataDirectory` | string | `"D:\\ISO11820\\TestData"` | 温度时序 CSV 数据目录 |

#### Report

| 键 | 类型 | 默认值 | 说明 |
|----|------|--------|------|
| `OutputDirectory` | string | `"D:\\ISO11820\\Reports"` | 报告输出目录 |
| `EnablePdfExport` | bool | `true` | 是否启用 PDF 导出 |

## 五、DbHelper 方法需求清单

根据数据流分析，`DbHelper` 类需封装以下操作：

| 类别 | 方法 | 对应 SQL | 调用时机 |
|------|------|---------|---------|
| **初始化** | `InitializeDatabase()` | CREATE TABLE × 6 + INSERT 初始数据 | 程序首次运行 |
| **登录** | `ValidateLogin(username, pwd)` → userid, usertype | SELECT from operators | 登录界面 |
| **设备** | `GetApparatus()` → Apparatus | SELECT from apparatus | 新建试验/主界面 |
| **设备** | `UpdateConstPower(apparatusId, power)` | UPDATE apparatus | 试验记录阶段 |
| **样品** | `InsertProduct(product)` | INSERT into productmaster | 新建试验 |
| **样品** | `GetProduct(productId)` → Product | SELECT from productmaster | 查询 |
| **样品** | `QueryProducts()` → List\<Product\> | SELECT from productmaster | 样品列表 |
| **试验** | `InsertTest(test)` | INSERT into testmaster（统计字段填 0） | 新建试验 |
| **试验** | `UpdateTestResult(productId, testId, result)` | UPDATE testmaster（统计字段 + flag） | 试验完成保存 |
| **试验** | `GetTest(productId, testId)` → Test | SELECT from testmaster | 查看详情 |
| **试验** | `QueryTests(from, to, productId?)` → List\<Test\> | SELECT from testmaster | 记录查询 |

| **校准** | `QueryCalibrations(from, to)` → List\<Calibration\> | SELECT from CalibrationRecords | 校准历史 |

## 六、数据约束与一致性规则汇总

| 编号 | 规则 | 影响 |
|------|------|------|
| R01 | `testmaster` 联合主键 `(productid, testid)` | 查询/更新必须同时提供两值 |
| R02 | `testmaster.productid` 外键引用 `productmaster.productid` | 必须先创建样品才能创建试验 |
| R03 | `testmaster` 新建时统计字段全部填 0 或 "" | 避免 NULL 干扰计算 |
| R04 | `flag = "10000000"` 表示试验记录已保存 | UI 状态控制的关键依据 |
| R05 | `totaltesttime > 0 且 flag ≠ "10000000"` 表示未保存 | 阻止新建试验、阻止记录 |
| R06 | 密码明文存储、operators 表无主键 | 登录查询只能按 username + pwd |
| R07 | `CalibrationRecords` 表名大写 | 注意 SQL 中大小写 |
| R08 | `TemperatureData` 字段存 JSON 字符串 | 需手动序列化/反序列化 |
| R09 | 温度时序数据不入库，存 CSV 文件 | 报告模块需独立读取 CSV |
| R10 | 程序首次运行时创建表结构 + 初始数据 | DbHelper 需判断是否已初始化 |
